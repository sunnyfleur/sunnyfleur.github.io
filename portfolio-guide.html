Ghi nhớ ngắn

Nếu bạn đang sửa nội dung project, gần như luôn luôn phải bắt đầu ở projects.json, không phải ở index.html.

3 quy tắc nhớ nhanh

Chỉ cần nhớ đúng 3 ý dưới đây là bạn đã tránh được phần lớn lỗi cập nhật nội dung.

1. Project sửa ở một nơi

Nội dung card project và nội dung trang chi tiết đều đọc từ projects.json.

2. project.html không phải nơi viết nội dung

File này chủ yếu là layout dùng chung. Chỉ sửa khi bạn muốn đổi cách hiển thị, không phải khi chỉ đổi text hay ảnh của một project.

3. Trước khi publish phải kiểm tra

Sau khi sửa xong, chạy powershell -ExecutionPolicy Bypass -File .\verify-portfolio.ps1 để kiểm tra cấu trúc cơ bản.

Cách sửa một project đang có

Đây là việc bạn sẽ làm nhiều nhất. Khi muốn sửa text, ảnh, role, gallery, link của một project, hãy làm theo đúng thứ tự dưới đây.

Bước 1. Mở đúng object trong projects.json

Tìm project bằng slug. Ví dụ: space-conqueror hoặc how-mob-feel.

Bước 2. Sửa phần nhìn thấy ngay trên homepage

Ba field quan trọng nhất cho homepage là title, tagline và summary.

Bước 3. Sửa ảnh đúng chỗ

thumbnail là ảnh card. heroImage là ảnh lớn ở đầu case study. gallery là danh sách ảnh bên trong project.

Bước 4. Sửa phần thông tin đọc sâu

Các phần như problem, contributions, systems và results là nội dung dùng cho trang chi tiết.

Ví dụ object rút gọn trong projects.json

Lưu ý rất quan trọng

Nếu bạn chỉ muốn đổi nội dung của project, đừng sửa trực tiếp card trong index.html. Project explorer mới không đọc card cứng ở đó nữa.

Cách thêm project mới

Muốn thêm project mới thì bạn không cần tạo thêm một HTML riêng. Chỉ cần thêm đúng data vào projects.json là homepage và trang chi tiết sẽ tự dùng lại layout có sẵn.

  1. Sao chép một object project đã có trong projects.json.
  2. Đổi slug thành tên mới, ngắn gọn và không trùng với project khác.
  3. Đổi title, tagline, summary và các phần text khác.
  4. Đặt ảnh mới vào img/ rồi cập nhật đường dẫn cho thumbnail, heroImage và gallery.
  5. Nếu muốn project nằm ở nhóm nổi bật, đặt featured thành true.
  6. Nếu muốn project xuất hiện khi lọc, thêm các tag phù hợp vào filters.
  7. Lưu file, mở index.html và project.html?slug=slug-cua-ban để xem lại.

Mẹo nhỏ

Nếu project mới chưa đủ tài liệu, hãy dùng cách viết trung thực kiểu archive: mô tả ngắn gọn, ảnh placeholder rõ ràng, không dùng text filler.

Cách sửa homepage

Những phần như giới thiệu cá nhân, about, resume, contact và metadata của trang chủ vẫn nằm trong index.html.

Hero / giới thiệu đầu trang

Vào index.html để sửa tên, headline, phần giới thiệu chính hoặc CTA ở khu vực đầu trang.

About Me

Nếu muốn cập nhật câu chuyện cá nhân, định hướng nghề nghiệp hoặc phong cách làm việc, sửa phần About trong index.html.

Resume / kinh nghiệm

Các mốc học tập, kinh nghiệm, kỹ năng và mô tả ngắn ở resume section đều nằm trong index.html.

Contact

Email, số điện thoại, link social, location và form contact cũng sửa trực tiếp ở index.html.

Nhớ phân biệt 2 loại nội dung

Project content nằm ở projects.json. Personal site content như about / resume / contact nằm ở index.html.

Đổi ảnh, CV và link

Đây là các thay đổi thường gặp nhưng dễ lỗi nếu quên cập nhật đường dẫn.

Đổi ảnh project

Thêm ảnh mới vào img/, sau đó sửa đường dẫn ở projects.json. Không chỉ chép ảnh vào thư mục rồi để nguyên path cũ.

Đổi CV

Thay file trong CV/, rồi kiểm tra các nút tải CV hoặc link đang trỏ đúng tới file mới.

Đổi link ngoài

Link project nằm trong field links của projects.json. Link social hoặc contact thì sửa trong index.html.

Đổi video

Nếu project có video, sửa field video trong projects.json. Nếu không có video, bạn có thể để trống.

Checklist trước khi publish

Đây là checklist ngắn nhất nên làm sau mỗi lần sửa nội dung.

1. Mở lại đúng trang cần xem Project thì xem ở project.html?slug=.... Trang chủ thì xem ở index.html.
2. Chạy script kiểm tra nhanh Dùng lệnh: powershell -ExecutionPolicy Bypass -File .\verify-portfolio.ps1
3. Kiểm tra ảnh có hiển thị đúng không Nếu ảnh mất, gần như chắc chắn là path sai hoặc file chưa nằm đúng thư mục.
4. Kiểm tra text ngắn trên homepage Nhìn lại title, tagline, summary vì đây là phần người xem đọc đầu tiên.
5. Kiểm tra CTA và link tải Nếu có nút download docs / CV / link build, bấm thử để chắc là không lỗi.
Lệnh kiểm tra nhanh
powershell -ExecutionPolicy Bypass -File .\verify-portfolio.ps1

Lỗi thường gặp và cách tránh

Nếu thấy project không hiện đúng hoặc trang nhìn kỳ lạ sau khi sửa, hãy kiểm tra các lỗi bên dưới trước.

  1. Sửa nhầm file: sửa nội dung project ở index.html thay vì projects.json.
  2. Slug bị trùng hoặc sai: làm URL project mở nhầm dữ liệu hoặc không hiện đúng project mong muốn.
  3. Sai đường dẫn ảnh: ảnh không hiện vì file chưa nằm đúng trong img/ hoặc path viết sai.
  4. Quên dấu phẩy / ngoặc trong JSON: lưu xong nhưng project explorer hoặc project page tải lỗi.
  5. Chỉ thay file CV mà không đổi link: nếu tên file đổi, các nút cũ sẽ trỏ sai.
  6. Summary quá dài: homepage sẽ khó scan. Hãy để summary rõ, gọn, đọc nhanh.

Nếu sau này workflow đổi thêm, hãy cập nhật file này cùng lúc với projects.json, project.html hoặc verify-portfolio.ps1 để tài liệu luôn khớp với cách làm thật.
